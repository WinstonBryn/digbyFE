REST API Version 1 for ForEver Application

Digital asset access

Creation and access of Digital assets
Resource Description
GET /api/v1/user/{id} This is used to retreive the information that is stored in the internal data structure of the forever application. The userId is the unique parameter that maps the user of digital asset.
POST /api/v1/user/post Not for invocation by the application but invoked from the digital asset when a new user is added to the application and the user has given consent to use the forever application.
PUT /api/v1/user/put Not for invocation by the application but invoked from the digital asset to forever digital asset.
DELETE /api/v1/user/delete/{id} Not for inovcation by the application but invoked from the digital asset when a digital asset is deleted.
ForEver application access

Creation and access of contacts sharing through ForEver Application
Resource Description
GET /api/v1/user/fe/adduser/get/{id} Returns the FE's asset of the user with the level of contacts shared.
POST /api/v1/user/fe/adduser/post Takes the FE's assets they are feName, feUserList, feGroupList. The feUserList has information about the users the contact information along with the level of sharing. The feGroupList has information about the group, the contact information with the level of sharing.
PUT /api/v1/user/fe/adduser/put Takes the FE's assets with the same data definition as that of POST. It removes the already stored information of the FE's asset and replaces with the request.
DELETE /api/v1/user/fe/adduser/delete/{id} Deleted the FE's asset of the user from the data structure.
ForEver application registration

Authentication of application access.
Resource Description
GET /api/v1/app/fe/addapp/get/{id} Returns the FE's token information used to share the application data.
POST /api/v1/app/fe/addapp/post Takes the FE's approval to obtain the digital asset information from DAG.
PUT /api/v1/app/fe/addapp/put Takes the FE's approval to obtain the digital asset information from DAG. It removes the already stored information of the approval and replaces with the current request.
DELETE /api/v1/app/fe/addapp/delete/{id} Deleted the FE's approval to obtain the digital asset information from DAG.
